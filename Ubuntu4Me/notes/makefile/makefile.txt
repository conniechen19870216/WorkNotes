1. Software
Overview: autoconf/automake/m4/perl/libtool(libtool not required)
(1) autoconf(m4):   自动配置软件包
(2) automake(perl): 将Makefile.am文件自动生成Makefile.in文件
(3) libtool:        便于生成各种程序库的工具
(4) autoheader:     助于生成config.h

2. automake 支持的三种目录层次
(1) flat:   所有文件都在同一个目录里面
(2) shadow: 主要源文件在当前目录，而其他的在不同目录里面
(3) deep:   所以源代码都在子目录里面，顶层目录只包含配置信息 

3. Procedures
(1) autoscan：autoscan.log/configure.scan
(2) mv configure.scan configure.in
    edit configure.in
(3) 在project目录以及core/shell目录下新建Makefile.am 
(4) 在project目录下新建NEWS, README, ChangeLog, AUTHOR
(5) 把/usr/share/automake-1,x目录下的depcomp和compile文件copy到project下
(6) aclocal
(7) autoconf
(8) automake --add-missing
(9) ./configure

4. Conclusions
(1) Create 3 Files: helloworld.c, configure.in, Makefile.am
(2) Procedures: aclocal; autoconf; automake --add-missing; ./configure; make; ./helloworld
(3) Actual Tools:
    autoscan: perl script
	aclocal:  perl script
	autoconf: shell script
	m4:       macro processor
	(autoconf uses m4 to process the aclocal.m4 generated by aclocal)
	automake: perl script

5. 标准安装路径
(1) Default: $(prefix) = /usr/local
(2) Overide: ./configure --prefix=<new_path>
(3) Other pre-defined path
    bindir = $(prefix)/bin, 
    libdir = $(prefix)/lib, 
	datadir = $(prefix)/share, 
	sysconfdir = $(prefix)/etc
